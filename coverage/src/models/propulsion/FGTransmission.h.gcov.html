      60             : /** Utility class that handles power transmission in conjunction with FGRotor.
      61             : 
      62             :   This class provides brake, clutch and free-wheel-unit (FWU) functionality
      63             :   for the rotor model. Also it is responsible for the RPM calculations.
      64             : 
      65             :   When the engine is off the brake could be used to slow/hold down a spinning
      66             :   rotor. The maximum brake power is defined in the rotors' config file.
      67             :   (Right now there is no checking if the input is in the [0..1] range.)
      68             : 
      69             :   The clutch operation is based on a heuristic approach. In the intermediate
      70             :   state the transfer is proportional to the clutch position. But equal RPM
      71             :   values are enforced on the thruster and rotor sides when approaching the
      72             :   closed state.
      73             : 
      74             :   The FWU inhibits that the rotor is driving the engine. To do so, the code
      75             :   just predicts the upcoming FWU state based on current torque conditions.
      76             : 
      77             :   Some engines won't work properly when the clutch is open. To keep them
      78             :   controllable some load must be provided on the engine side (EngineFriction,
      79             :   aka gear-loss). See the notes under 'Engine issues' in FGRotor.
      80             : 
      81             : <h3>Property tree</h3>
      82             : 
      83             :   The following properties are created (with x = your thruster number):
      84             :   <pre>
      85             :     propulsion/engine[x]/brake-ctrl-norm
      86             :     propulsion/engine[x]/free-wheel-transmission
      87             :     propulsion/engine[x]/clutch-ctrl-norm
      88             :   </pre>
      89             : 
      90             : <h3>Notes</h3>
      91             : 
      92             :   <ul>
      93             :     <li> EngineFriction is assumed constant, so better orientate at low RPM
      94             :          values, because piston and turboprop engines don't 'like' high
      95             :          load at startup.</li>
      96             :     <li> The model doesn't support backward operation.</li>
