      56             : /** Models atmospheric disturbances: winds, gusts, turbulence, downbursts, etc.
      57             : 
      58             :     <h2>Turbulence</h2>
      59             :     Various turbulence models are available. They are specified
      60             :     via the property <tt>atmosphere/turb-type</tt>. The following models are
      61             :     available:
      62             :     - 0: ttNone (turbulence disabled)
      63             :     - 1: ttStandard
      64             :     - 2: ttCulp
      65             :     - 3: ttMilspec (Dryden spectrum)
      66             :     - 4: ttTustin (Dryden spectrum)
      67             : 
      68             :     The Milspec and Tustin models are described in the Yeager report cited
      69             :     below.  They both use a Dryden spectrum model whose parameters (scale
      70             :     lengths and intensities) are modelled according to MIL-F-8785C. Parameters
      71             :     are modelled differently for altitudes below 1000ft and above 2000ft, for
      72             :     altitudes in between they are interpolated linearly.
      73             : 
      74             :     The two models differ in the implementation of the transfer functions
      75             :     described in the milspec.
      76             : 
      77             :     To use one of these two models, set <tt>atmosphere/turb-type</tt> to 4
      78             :     resp. 5, and specify values for
      79             :     <tt>atmosphere/turbulence/milspec/windspeed_at_20ft_AGL-fps</tt> and
      80             :     <tt>atmosphere/turbulence/milspec/severity</tt> (the latter corresponds to
      81             :     the probability of exceedence curves from Fig.&nbsp;7 of the milspec,
      82             :     allowable range is 0 (disabled) to 7). <tt>atmosphere/psiw-rad</tt> is
      83             :     respected as well; note that you have to specify a positive wind magnitude
      84             :     to prevent psiw from being reset to zero.
      85             : 
      86             :     Reference values (cf. figures 7 and 9 from the milspec):
      87             :     <table>
      88             :       <tr><td><b>Intensity</b></td>
      89             :           <td><b><tt>windspeed_at_20ft_AGL-fps</tt></b></td>
      90             :           <td><b><tt>severity</tt></b></td></tr>
      91             :       <tr><td>light</td>
      92             :           <td>25 (15 knots)</td>
      93             :           <td>3</td></tr>
      94             :       <tr><td>moderate</td>
      95             :           <td>50 (30 knots)</td>
      96             :           <td>4</td></tr>
      97             :       <tr><td>severe</td>
      98             :           <td>75 (45 knots)</td>
      99             :           <td>6</td></tr>
     100             :     </table>
     101             : 
     102             :     <h2>Cosine Gust</h2>
     103             :     A one minus cosine gust model is available. This permits a configurable,
     104             :     predictable gust to be input to JSBSim for testing handling and
     105             :     dynamics. Here is how a gust can be entered in a script:
     106             : 
     107             :     ~~~{.xml}
     108             :     <event name="Introduce gust">
     109             :       <condition> simulation/sim-time-sec ge 10 </condition>
     110             :       <set name="atmosphere/cosine-gust/startup-duration-sec" value="5"/>
     111             :       <set name="atmosphere/cosine-gust/steady-duration-sec" value="1"/>
     112             :       <set name="atmosphere/cosine-gust/end-duration-sec" value="5"/>
     113             :       <set name="atmosphere/cosine-gust/magnitude-ft_sec" value="30"/>
     114             :       <set name="atmosphere/cosine-gust/frame" value="2"/>
     115             :       <set name="atmosphere/cosine-gust/X-velocity-ft_sec" value="-1"/>
     116             :       <set name="atmosphere/cosine-gust/Y-velocity-ft_sec" value="0"/>
     117             :       <set name="atmosphere/cosine-gust/Z-velocity-ft_sec" value="0"/>
     118             :       <set name="atmosphere/cosine-gust/start" value="1"/>
     119             :       <notify/>
     120             :     </event>
     121             :     ~~~
     122             : 
     123             :     The x, y, z velocity components are meant to define the direction vector.
     124             :     The vector will be normalized by the routine, so it does not need to be a
     125             :     unit vector.
     126             : 
     127             :     The startup duration is the time it takes to build up to full strength
     128             :     (magnitude-ft_sec) from zero. Steady duration is the time the gust stays at
     129             :     the specified magnitude. End duration is the time it takes to dwindle to
     130             :     zero from the specified magnitude. The start and end transients are in a
     131             :     smooth cosine shape.
     132             : 
     133             :     The frame is specified from the following enum:
     134             : 
     135             :     enum eGustFrame {gfNone=0, gfBody, gfWind, gfLocal};
     136             : 
     137             :     That is, if you specify the X, Y, Z gust direction vector in the body frame,
     138             :     frame will be "1". If the X, Y, and Z gust direction vector is in the Wind
     139             :     frame, use frame = 2. If you specify the gust direction vector in the local
     140             :     frame (N-E-D) use frame = 3. Note that an internal local frame direction
     141             :     vector is created based on the X, Y, Z direction vector you specify and the
     142             :     frame *at the time the gust is begun*. The direction vector is not updated
     143             :     after the initial creation. This is to keep the gust at the same direction
     144             :     independent of aircraft dynamics.
     145             : 
     146             :     The gust is triggered when the property atmosphere/cosine-gust/start is set
     147             :     to 1. It can be used repeatedly - the gust resets itself after it has
     148             :     completed.
     149             : 
     150             :     The cosine gust is global: it affects the whole world not just the vicinity
     151             :     of the aircraft.
     152             : 
     153             :     @see Yeager, Jessie C.: "Implementation and Testing of Turbulence Models for
     154             :          the F18-HARV" (<a
     155             :          href="http://ntrs.nasa.gov/archive/nasa/casi.ntrs.nasa.gov/19980028448_1998081596.pdf">
     156             :          pdf</a>), NASA CR-1998-206937, 1998
     157             : 
     158             :     @see MIL-F-8785C: Military Specification: Flying Qualities of Piloted Aircraft
     159             : 
     160             : */
