      95             : /** Encapsulates code that models an individual arbitrary force, moment or a
      96             :     combination thereof.
      97             :     This class encapsulates an individual reaction applied at the specified
      98             :     location on the vehicle, and oriented as specified in one of three frames:
      99             : 
     100             :     - BODY frame is defined with the X axis positive forward, the Y axis
     101             :            positive out the right wing, and the Z axis completing the set
     102             :            positive downward out the belly of the aircraft.
     103             :     - LOCAL frame is a world-based frame, with X positive north, Y positive east
     104             :             and Z completing the right handed system positive down towards
     105             :             the center of the Earth.
     106             :     - WIND frame (rotated) has X negative into the wind vector (in other words
     107             :            drag is along the positive X axis), the Z axis is perpendicular to
     108             :            X and positive up (lift) but in the aircraft XZ plane, and Y
     109             :            completes the right handed system. This is modified from a normal
     110             :            wind frame definition, which is rotated about the Y axis 180 degrees
     111             :            from this WIND frame.
     112             : 
     113             :     Much of the substance of this class is located in the FGForce base class,
     114             :     from which this class is derived.
     115             : 
     116             :     Here is the XML definition of a force (optional items are in []):
     117             : 
     118             :     @code
     119             :     <force name="name" frame="BODY | LOCAL | WIND">
     120             : 
     121             :       [<function> ... </function>]
     122             : 
     123             :       <location unit="{IN | M}"> 
     124             :         <x> {number} </x>
     125             :         <y> {number} </y>
     126             :         <z> {number} </z>
     127             :       </location>
     128             :       [<direction> <!-- optional initial direction vector -->
     129             :         <x> {number} </x>
     130             :         <y> {number} </y>
     131             :         <z> {number} </z>
     132             :       </direction>]
     133             :     </force>
     134             :     @endcode
     135             : 
     136             :     The location of the force vector, in structural coordinates, can be set at
     137             :     runtime through the following properties:
     138             : 
     139             :     @code
     140             :     external_reactions/{force name}/location-x-in
     141             :     external_reactions/{force name}/location-y-in
     142             :     external_reactions/{force name}/location-z-in
     143             :     @endcode
     144             : 
     145             :     The XML definition of a moment (optional items are in []) is a bit simpler
     146             :     because you do not need to specify the location:
     147             : 
     148             :     @code
     149             :     <moment name="name" frame="BODY | LOCAL | WIND">
     150             : 
     151             :       [<function> ... </function>]
     152             : 
     153             :       [<direction> <!-- optional initial direction vector -->
     154             :         <x> {number} </x>
     155             :         <y> {number} </y>
     156             :         <z> {number} </z>
     157             :       </direction>]
     158             :     </moment>
     159             :     @endcode
     160             : 
     161             :     The initial direction can optionally be set by specifying a unit vector
     162             :     in the chosen frame (body, local, or wind).
     163             : 
     164             :     As an example, a parachute can be defined oriented in the wind axis frame
     165             :     so the drag always acts in the drag direction - opposite the positive X
     166             :     axis. That does not include the effects of parachute oscillations, but
     167             :     those could be handled in the calling application.
     168             : 
     169             :     The force (or moment) direction is not actually required to be specified as
     170             :     a unit vector, but prior to the force (or moment) vector being calculated,
     171             :     the direction vector is normalized when initialized.
     172             : 
     173             :     The force direction can be specified at runtime through setting any/all of
     174             :     the following properties:
     175             : 
     176             :     @code
     177             :     external_reactions/{force name}/x
     178             :     external_reactions/{force name}/y
     179             :     external_reactions/{force name}/z
     180             :     @endcode
     181             : 
     182             :     The moment direction can be specified at runtime through setting any/all of
     183             :     the following properties:
     184             : 
     185             :     @code
     186             :     external_reactions/{moment name}/l
     187             :     external_reactions/{moment name}/m
     188             :     external_reactions/{moment name}/n
     189             :     @endcode
     190             : 
     191             :     However in that case, the direction is no longer normalized.
     192             : 
     193             :     When no <function> has been provided in the force definition, its magnitude
     194             :     can be specified through the following property:
     195             : 
     196             :     @code
     197             :     external_reactions/{force name}/magnitude
     198             :     @endcode
     199             : 
     200             :     When no <function> has been provided in the moment definition, its magnitude
     201             :     can be specified through the following property:
     202             : 
     203             :     @code
     204             :     external_reactions/{moment name}/magnitude-lbsft
     205             :     @endcode
     206             : */
